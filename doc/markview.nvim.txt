‚ú® Features                                              *markview.nvim-feature*

Core features,

‚óá Supports previewing `HTML`, $LaTeX$, `Markdown`, `Typst` & `YAML`.
‚óá Highly customisable! Everything is done via the configuration table to ensure
  maximum customisability.
‚óá Hybrid editing mode! Allows editing & previewing files at the same time.
‚óá Split view! Allows previewing files on a separate window that updates in
  real-time!
‚óá Previews are compatible with `'wrap'`(only for `markdown` at the moment).
‚óá Dynamic config that allows any option to be a function.
‚óá Dynamic `highlight groups` that automatically updates with the colorscheme.

‚óá Supports heading elements out of the box.

ü™≤ Known bugs

‚óÜ `code span`s don't get recognized when on the line after a `code block`(if the
  line after the `code span` is empty).
  This is most likely due to some bug in either the `markdown` or the
  `markdown_inline` parser.

‚óÜ Incorrect wrapping when setting `wrap` using `modeline`.
  This is due to `textoff` being 0(instead of the size of the `statuscolumn`)
  when entering a buffer.

üìû Autocmds                                             *markview.nvim-autocmds*

`markview.nvim` emits various autocmd events during different parts of the
rendering allowing users to extend the plugin's functionality.

>lua
	vim.api.nvim_create_autocmd("User", {
	    pattern = "MarkviewAttach",
	    callback = function (event)
	        --- This will have all the data you need.
	        local data = event.data;
	
	        vim.print(data);
	    end
	})
<

üåü Presets                                               *markview.nvim-presets*

‚ñã Ó≠Ç Important
‚ñã Presets are looking for contributors!
‚ñã
‚ñã If you have any custom configuration that you would like to have as a preset
‚ñã you can open a `pull request` for that.

Check the wiki page {9} for more information.

USAGE

>lua
	local presets = require("markview.presets");
	
	require("markview").setup({
	    markdown = {
	        headings = presets.headings.slanted
	    }
	});
<


Currently available presets are,

HEADINGS

Accessed using `require("markview.presets").headings`.

‚óÜ `glow`
  Like the headings in the `Glow` CLI app.

‚óÜ `glow_center`
  Centered version of `glow`.

‚óÜ `slanted`
  Heading level + slanted tail.

‚óÜ `arrowed`
  Heading level + arrowed tail.

‚óÜ `simple`
  Headings similar to headings in some books.

‚óÜ `marker`
  Simple marker for heading level.

HORIZONTAL_RULES

Accessed using `require("markview.presets").horizontal_rules`.

‚óÜ `thin`
  Simple line.

‚óÜ `thick`
  Slightly thicker line.

‚óÜ `double`
  Double lines.

‚óÜ `dashed`
  Dashed line.

‚óÜ `dotted`
  Dotted line.

‚óÜ `solid`
  Very thick line.

‚óÜ `arrowed`
  Arrowed line.

TABLES

Accessed using `require("markview.presets").tables`.

‚óÜ `none`
  Border-less table(kinda like the ones used in some help files).

‚óÜ `single`
  Single border table.

‚óÜ `double`
  Double border table.

‚óÜ `rounded`
  Single border with rounded edges(default).

‚óÜ `solid`
  Solid border table.

üéÅ Extra modules                                          *markview.nvim-extras*

`markview.nvim` comes with a few extra things for added functionality. Currently
available modules are,

‚óÜ checkboxes.lua {11}
  Checkbox toggle, state change & more! Supports visual mode too with checkbox
  state caching!
  Wiki section {10}

‚óÜ editor.lua {13}
  A simple `code block` creator & editor with support for nested elements!
  Wiki section {12}

‚óÜ headings.lua {15}
  Simple heading level changer with support for `setext` headings.
  Wiki section {14}

Example usage,

>lua
	-- Load the checkboxes module.
	require("markview.extras.checkboxes").setup();
<


Go over a line with a checkbox and run,

>vim
	:Checkbox interactive
<


Now use `h`, `j`, `k`, `l` to change the checkbox state.
